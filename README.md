# AI-Powered Dominoes Game

A desktop-based dominoes game that combines artifiial intelligence and a classic tabletop game, creating a simple user-friendly game GUI that aims to create a fun and challenging game utilizing search algorithms and machine learning models.

---

## 📖 Project Background

The core motivation behind this project is to implement both machine learning and search algorithms in games, which is done in order to create a more fluid and challenging game for players. 

Instead of relying purely on fixed, static rules, this project implements two approaches to game AI:
1. **Search Algorithm**: Classic tree-traversal techniques suh as DFS, BFS, and A-Star search that compute future possibilities to find an optimal move.
2. **Imitation Learning**: Machine learning classifiers trained on data generated by expert matches between search algorithms. By studying these generated games, the models memorize and learn patterns which results in the best possible move against an opponent.

---

## 🎮 Game Features & GUI Functionalities

### 1. User Profiles & Dynamic Session Configuration
* **Identity Tracking**: Players can register or log in with a unique nickname, allowing the game to store a player's best game against specific algorithms or models.
* ** Opponent Swapping**: Players can switch out their computer opponent at the launch screen or instantly upon game completion (win, loss, or tie), allowing users to test or play different strategies against different AIs with dfferent approaches to learning.

### 2. Simple Interaction Controls
* **Accidental Click Prevention**: To prevent misclicks, a domino tile can only be committed to the board by mouse double-click.
* **Drag-and-Drop with Gesture Cancel**: Players can drag and drop tiles into the board and cancel said moves by simply dropping the tile back near their hand rack, snapping the domino grid cleanly back into place.

### 3. Layout Rendering
* **Automated Snaking**: Prevents domino trains from being too big and running off the edge of the window by tracking tile sequences, where the train would shift down after six consecutive tiles are played.
* **Live Match Clock**: A timer which tracks a player's elapsed match length, which calculates how long it takes for a player to beat an algorithm, counting only the player's time taken to select tiles, and would stop only when a terminal state (win, loss, or tie) is hit.

### 4. Competitive Leaderboard
* **Multi-Window Access**: The leaderboard can be reviewed either from the login window or the active game screen, only when the match is not running.
* **Performance Ranking**: Player rankings  are pulled from historical records and dynamically sorted through a dual-priority key: **Total Wins (Highest First) ➔ Fastest Completion Time (Lowest First)**.

## 🧠 Supported AI Opponents & Models

### 1. Search Algorithms
Compute optimal paths by calculating combinations based on game rules and positional advantages:
* **Breadth-First Search (BFS) Bot**: Analyzes all immediate valid plays, evaluating the board surface to maximize immediate card depletion.
* **Depth-First Search (DFS) Bot**: Explores narrow, highly targeted paths down specific tile suits, attempting to run players out of matching options early on.
* **A\* Search Bot**: Combines historical move counts with an advanced, look-ahead pip penalty heuristic. Prioritizes dumping heavy tile weights (like 6|6) early and tracks past "passes" (when a player skips their turn) to purposefully starve the human player of matching cards.

### 2. Machine Learning Ensembles
These bots utilize pattern recognition by extracting features from the current board (such as hand sizes, unplayed suit distributions, and player vulnerabilities) and choosing the move that scores the highest winning probability based on their training data:
* **Random Forest Bot**: Utilizes an ensemble of independent decision trees to vote on the most tactically sound move given the immediate arrangement of cards.
* **XGBoost Bot**: Employs a sequentially optimized, gradient-boosted decision tree framework that corrects predictive errors to mimic master-level execution speeds in milliseconds. counts with an advanced, look-ahead pip penalty heuristic.
